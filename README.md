## Installation

### From source

```bash
cd speckit-to-issue
pip install -e .
```

### Requirements

- Python 3.11+
- [GitHub CLI (gh)](https://cli.github.com) installed and authenticated

## Usage

### Create issues from tasks

```bash
# Basic usage
speckit-to-issue create specs/001-feature/tasks.md

# Dry run (preview only)
speckit-to-issue create specs/001-feature/tasks.md --dry-run

# Skip completed tasks
speckit-to-issue create specs/001-feature/tasks.md --skip-complete

# Format for Copilot Coding Agent
speckit-to-issue create specs/001-feature/tasks.md --assign-copilot

# Target specific repository
speckit-to-issue create specs/001-feature/tasks.md --repo owner/repo
```

### Check sync status

```bash
speckit-to-issue status specs/001-feature/tasks.md
```

### Show version

```bash
speckit-to-issue --version
```

## CLI Options

### `create` command

| Option | Short | Description |
|--------|-------|-------------|
| `--dry-run` | `-n` | Preview without creating issues |
| `--skip-complete` | `-s` | Skip tasks marked as complete |
| `--assign-copilot` | `-c` | Format issues for Copilot Coding Agent |
| `--force` | `-f` | Create issues even if they already exist |
| `--repo` | `-r` | Target repository (owner/repo format) |
| `--milestone` | `-m` | Milestone to assign issues to |
| `--verbose` | `-v` | Show detailed output |

### `status` command

| Option | Short | Description |
|--------|-------|-------------|
| `--repo` | `-r` | Target repository (owner/repo format) |
| `--verbose` | `-v` | Show detailed output |

## Tasks.md Format

The parser expects tasks in this format:

```markdown
## Phase 1: Setup

### T001: Create project structure
- **Priority:** Must
- **Estimate:** 10 min
- **Dependencies:** None
- **File:** `src/main.py`
- **FR:** FR-001
- **Acceptance Criteria:**
  - [ ] Create src/ directory
  - [ ] Add __init__.py file
```

### Completion Detection

Tasks are marked complete when:
- Title contains ‚úÖ emoji: `### T001: Task title ‚úÖ`
- All acceptance criteria are checked: `[x]`

## Generated Labels

The tool automatically creates and assigns labels:

| Label | Color | Description |
|-------|-------|-------------|
| `task` | Blue | All speckit tasks |
| `speckit` | Purple | Generated by speckit-to-issue |
| `priority:high` | Red | Must have |
| `priority:medium` | Yellow | Should have |
| `priority:low` | Green | Could have |
| `phase-N` | Purple | Phase number |
| `spec:name` | Light blue | Spec name |

## Troubleshooting

### "gh: command not found"

Install GitHub CLI from https://cli.github.com

### "Not logged in to any GitHub hosts"

Run `gh auth login` and complete authentication

### "Repository not found"

Check repository name or ensure you have access

### "Rate limit exceeded"

Wait a few minutes or check `gh api rate_limit`
